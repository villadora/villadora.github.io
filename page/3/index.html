泡沫：财富转移的工具

很多媒体都说经济泡沫多么可怕，经济在泡沫之后会经历巨大衰退，所有人的财富都会缩水，大家生活质量都会下降，是对所有人不利的；并且在各种泡沫中最喜欢用的词就是蒸发了多少市值，多少财富消失了；然而，事实真是如此嘛？

让我们仔细看看，财富的定义是什么。财富只是钱嘛？有人有10w块钱，有人只有1w块，加上一所100w的房子。论钱来说，第一个人更有“钱”，但现实中，大家都会认为第二个人更富有；那么财富就是固定资产么? 有人有100w的房子，有人拥有一家年利润500w的企业的全部股权，大家会觉得谁更有财富呢？现实中大家还是会选择第二个人。那么财富到底是什么，其实只要能够用来交换你需要的物质的东西都是你的财富，钱被人们认为是财富是因为钱具有最高的流动性，几乎所有人都接受现金支付。而房子企业这些都是可以交换来钱的东西，或者也可以直接去交换自己想要的东西，如吃的玩的。而有些时候，权力能够让人在不付一分钱的时候享受，换来自己所需的物质，所以某些时候权力也是财富的一种。

这个世界上的经济泡沫很多，认购券泡沫，股市泡沫，房产泡沫等等。那么泡沫破灭的时候什么东西变了呢？以前标价100元的股票现在只有10元了，以前300万的房子现在只有150万了，但是股票还是那个股票，房子还是那个房子，并不会因为股票面值变低了，企业的规模也缩小了（有时候企业甚至还在快速扩张），房子也不会因为价值的降低而面积变小了。如果有人在泡沫发生前就以10元拥有了股票，那么在结束时他的财富没有变化的，只要他没有参与泡沫的形成和崩溃过程中；而如果他在泡沫前以5元拥有了股票，那么在泡沫结束时他的财富用来计算还是增长的。也就是说对于没有参加泡沫整个过程的人来说，其财产并没有收到影响。期间变化的只不过是他财产的纸面价值罢了。有一群书法爱好者，其中一个人把自己的书法标价100万元，虽然实际上只有100元不到，但是他可以说自己拥有了100万元的财产，和泡沫过程中的人们一样；你可以说这是疯子，没有任何意义，因为没有任何交易可以支持这个价格。如果说另一个书法爱好者把自己的书法标价50万元，然后用2副自己的书法换了100万元的那幅，那么是不是这两个人都有了100万的财富呢？当然大家还是觉得这比较荒谬，现在这群书法爱好者都将自己的书法价格提高一万倍进行相互交换，并且聘请专业的会计和评估公司做操作，一切都合法合理了。因为这个故事太荒谬，所以大家都在嘲笑他们，而没有外面人真正的花真金白银去购买他们的书法。这样一个价格严重偏离实际价值的资产是不是泡沫呢？那么你认为这群书法爱好这的财富增长了么？有一天忽然大家说不玩了，又回归了原来的价格，那么这个泡沫是不是破灭了，他们的财产是不是缩水了呢？
如果你认为他们的财富增长了，那么最后他们的财富也就在泡沫破灭后缩水了；然而现实中，如果我说我写的字值100万，而我自己在家里写了10张，你是否会同样的判断说我就已经是千万富翁了呢？
而在泡沫中感到自己的资产升值了的人都是被市场上的价格而不是价值蒙蔽了双眼，没有套现就实际上没有实现自己的财富增值。因为不管我怎么标价，我写的字就是一文不值，一旦脱离了我自己的家，就要价值回归。而书法爱好者们高额标价的书法一旦脱离了他们那个圈子也就要实现价值回归。如果全社会的人都参与了这个泡沫呢？只要永远都有足够的资金，所有人都不抽身而去，那么这个泡沫就会永远的持续下去；只不过这两个前提永远都没有实现过。

既然泡沫的过程中没有参与泡沫的人的资产并没有任何变化，那么我们来看看参与了泡沫的过程人；在泡沫中买入标的物的人，实际上是付出了自己的财产，这些财产被卖出者拥有；一个人花100万从另一个人手中买了房子，100万没有变，房子也没有边，边的只不过是钱和物都易主了。如果卖出房产的人不再将100万投入房产市场，而买房子的人也不再在泡沫过程中卖出房产；那么之后如果房产涨到300万再跌到50万，无论是卖出还是买入者，自身的财产都不在受到影响。买入者无论涨跌多少都是拥有一套房子，卖出者无论涨跌多少都只拥有100万现金。双方财富总和也没有变，只不过发生了交换罢了。
A以5元一股买入10w股票，然后在股市泡沫涨到20元时卖给B，B付出了200万元给A，B买到股票后泡沫破灭，股票又下跌到5元。我们来看下这个过程中是怎样的:

+————+————-+———-+———-+———————+
| | A | B | 市场 | 总共 |
+————+————-+———-+———-+———————+
|初始 |50万元 |200万元| 10万股|250万元+10万股|
+————+————-+———-+———-+———————+
|A买入 |10万股 |200万元| 50万元|250万元+10万股|
+————+————-+———-+———-+———————+
|B买入 |200万股 |10万股 | 50万元|250万元+10万股|
+————+————-+———-+———-+———————+
|泡沫破灭|200万股 |10万股 | 50万元|250万元+10万股|
+————+————-+———-+———-+———————+

可以看到，在整个过程中社会上的总财富没有任何变动，变化的只不过是A的50万元流入了市场中其他人的口袋，B的200万元流入了A的口袋。这中间没有劳动没有创造，交易是不会产生任何财富的，但是交易去可以用来转移财富实现剥削。

把泡沫破灭是所有人的损失的特点是舆论的造势，让人们恐惧泡沫的破灭，让人们自愿的承受本来应该由那些泡沫的制造者和参与者承受的损失，让人们去投入自己财富去维持泡沫。次贷危机之前，华尔街的银行家和雇员们拿的高额薪水酬劳是不会吐出来的，股票市场中通过高额溢价圈钱实现一夜暴富的富翁们的钱也不会换回来的。他们在泡沫中赚的金银满盆，真是因为他们的高额收入，所以才会有泡沫之后大面积的损失，因为他们的所得已经超过平均回报，1个人赚的钱必须有100个人的损失来承担；实际上就是利用泡沫剥削了那100个参与者。

每次泡沫结束后，都有着一大群的蛀虫已经不再需要任何劳动就能够享受人生了，而他们获取之多，所推高的资产价格甚至已经超过了社会的总财富。所以当人们恐惧泡沫的破灭恐惧经济GDP增长率在数字上的衰退而继续维持资产价格时，经济增长智能停滞，知道人们的辛勤劳动填补泡沫形成的空洞位置。也就是说股票泡沫，企业的员工需要付出更多劳动为企业增加盈余，使得企业最终的生产力和资本慢慢符合股票的价位；房地产泡沫，则需要人们付出更多的劳动，使得该块土地上的劳动生产率可以支持房地产的价格，也就是能够支持合理回报的租金；而这些额外的劳动价值在泡沫发生的时候就已经被那些从泡沫中获利的人给透支了。

泡沫从来没有导致财富的产生，其破灭也不会导致财富的消亡，只不过让财富秘密的集中，形成少数人的富裕。

从程序员角度来看历史(一)

一、专业化与冗余

经济是政治社会制度的基础，时代的经济情况如何决定了该时代会有怎样的政治制度。人类社会的生产形式一直是从多样化走向专业化的，古代全社会绝大部分都是农民，因为吃饭是最大的问题，生产力不够，只能让人来生产保障吃的吧。同时，有一部分人分离了出来专门从事记事发展文化之类，开始还是兼职，读书人和庄稼人身份之间还是模糊的，在夏商周的时代，有些贵族也是从事这农活水利等方面的活动的，当然，逐渐的由一小部分变成不再从事相关的工作。最后，终于有一部分人从马斯洛需求的第一第二层次中摆脱了出来，不用在为生存和安全操心，可以全心全意的投入到其他工具的开发，如组织，调度，使得人能够以集体的方式完成对自然的大规模改造，也同时开始了人类集体的进化。

从程序的角度来说，就是最开始的要求只有functional requirements的实现，其余的performance, scalability, extendibility之类，只有在functional requirement能够实现的基础上才有意义。

而简单的程序最初也是一个component完成所有的工作，计算机最早的阶段一个程序就是一盒磁带，不存在条件实现模块化等分割。

在汉代，随着人们对时令气候等逐渐了解，由有部分人解放了出来，可以从事捕鱼，伐木，炼铁等。专业化导致人们的技能在不断的练习中得到提高，这样就提高了生产力；而提高生产力的结果就是把更多的人从第一第二层的需求中解脱出来，而发明与工具的改进又进一步提高了生产力，从而是人可以从事更高需求的活动。

即使在农业其本身，专业化的过程也在继续；从一开始的人们要耕地插秧播种收割都包办，到开始在家庭中男女之间进行了分工；之后就是在一个更大的集体，比如家族村庄中进行分工，有人负责赶牛拉犁，有人负责在后面插秧。

对于一个程序来说，就是说分化出更多的模块，有的负责计算，有的负责处理用户输入，有的负责与IO的打交道之类。每个部分做自己专门的事情，代码变得清晰，同时由于更加专业化同质化，更容易做优化，也更容易做指令或者执行过程的调度，从而和社会中生产会提高效率一样，程序的专业化也会提高效率。

总而言之，专业化的最终目的就是：效率。

之后的发展除了再继续更细化的分工之外，就是规模化和集成化了。大政府大企业的存在和技术的进步实现了前人无法现象的规模化生产，而规模效应直到亚当斯密在《国富论》中做了揭示之后才慢慢从实践的道路走向理论的道路上来。而之后才由阿尔弗雷德.马歇尔将之变成科学的研究学科，并由亨利.福特，规模生产的疯狂崇拜者，将其推广实践推向巅峰。

对于读书与分析问题的几个观点

读书也好分析问题也好，一个人的思维总归是被某种观念所占据，其行动也是被这种观点所指引；如读书，学习分析问题，这是改进和断念其指导思想的方法。我想谈下在这个过程中所需要注意的几点：

  1. 尽信书不如无书。
    自己不好好思考不动脑看到是什么就是什么的人参与政治和历史这种研究主体是人的问题很容易被人忽悠；因为这些东西比较主观，很难让人从客观的角度来审视；一般不会有绝对的对错，所以如果相信书上或者某些人说的就一定是正确的，而一定要去争个面红耳赤的还是研究客观的问题好。

  2. 发展的眼光看问题。
    很不幸，目前来看我们所处的宏观世界里时间只能向前，过去的合适并不一定现在合适，现在合适的并不一定过去合适。钱穆先生在书中说的好，一个是历史意见，一个是时代意见。我们不应该单凭时代意见来抹杀已往的历史意见。过去的就是历史，只有当时当地所出的环境下的当事人意见才是真实有效的，事后诸葛亮来指点着指点那，就应该当作是学习经验，而不是指手画脚。而以时代意见指导历史意见与刻舟求剑何异？一个人的行动到底是由思想所指导还是由习惯所指导将决定其是否能适应变化。

  3. 全面的考虑。
    从片面的条件推导出结论，如果不是真的疏忽和考虑欠缺，而是有意为之，要就只能是别有用心了。屁股决定脑袋这个道理无往不利，处于什么地位处于哪个阶级自然就要或多或少为自己考虑，所不同的是考虑长期还是短期。而我们已知的事件都是过往的人和事，除非是要利用历史的解读来为自己的利益谋取理由，否则应该能够脱离当事人的局限，从更大的角度来看历史事件。这样才能够有所学习和进步。当然为自己的利益而进行有选择性的注解事实并不是不可以的，功利主义对于现在更有意义，即使对于过去和未来并非如此。

虽然有很多优势，但是总体上来看还是巧妙的利用了移动设备的局限性：屏幕小；不可能同时有多个app同时在前台，实际上iphone上来说更加简化，屏幕图像渲染永远都只有一个app,提供的7个多任务API中根本就没有关于图像重绘的，所以即使显示了多个窗口也最多是系统保留切换前最后一刻的画面。所以iphone os的multitasking 能做到节省电能，不影响性能同时方便切换。

Smart Pointer

看了下Smart Pointer的实现，其实是对常规指针进行了封装

牵涉到两个问题:

  • 1) 保证对象的唯一所有权，也就是说对于一个ptr，同一时刻只会有一个Smart Pointer拥有.

如smart pointer的一个构造函数的实现:

拷贝构造函数调用了 =__a= 的release方法， 所以实际上指针在两个 =auto_ptr= 传递了
在其他过程中也是这样，始终保持只有一个 =auto_ptr= 对 =_M_ptr= 保持所有权

  • 2) 生命周期控制

=auto_ptr= 是一个构建在栈上的object，这样当退出代码区时，程序会自动释放掉栈上的 =auto_ptr=，而 =auto_ptr= 的析构函数则调用了delete销毁了分装的 =_M_ptr= 指针

SSA note

Static Single Assignment, dominate frontier, dominate basic block

SSA From&def-use chain on wiki, Efficiently Computing Static Single Assignment Form and Control Dependence Graph 91’

在gcc内部主要是在tree，gimple阶段，进行了dead code remove, use-def/def-use analysis, 以及许多在tree形态下需要多项式来完成的优化，在转化为ssa后，由于varriables都只有一个defination，所以只需要线性时间

gcc中ssa ir和tree是同时存在的，在转化为rtl后，就不存在ssa结构了，在regrename pass中，倒是对每一条insn，做了分析，由build_def_use function 生成def-use chain，

主要流程:

  • recog_insn_type
  • extract_insn
    • insn_extract
    • fill the recog_data, including operand nums, operand types and operand modes.
  • prepare_constraints
    • check each operands
  • scan_rtx
    • scan_rtx, if it’s insn type, call scan_rtx recursively, if it’s reg, call scan_rtx_reg, if it’s mem, call scan_rtx_address.
    • scan_rtx_reg, scan_rtx_address, check every rtx ( most are regs ), add new or close old chains created in previous stage.
