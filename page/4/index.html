Smart Pointer

看了下Smart Pointer的实现，其实是对常规指针进行了封装

牵涉到两个问题:

  • 1) 保证对象的唯一所有权，也就是说对于一个ptr，同一时刻只会有一个Smart Pointer拥有.

如smart pointer的一个构造函数的实现:

拷贝构造函数调用了 =__a= 的release方法， 所以实际上指针在两个 =auto_ptr= 传递了
在其他过程中也是这样，始终保持只有一个 =auto_ptr= 对 =_M_ptr= 保持所有权

  • 2) 生命周期控制

=auto_ptr= 是一个构建在栈上的object，这样当退出代码区时，程序会自动释放掉栈上的 =auto_ptr=，而 =auto_ptr= 的析构函数则调用了delete销毁了分装的 =_M_ptr= 指针
套用别人的原话：利用栈上对象的生命期控制堆上对象的生命期

SSA note

Static Single Assignment, dominate frontier, dominate basic block

SSA From&def-use chain on wiki, Efficiently Computing Static Single Assignment Form and Control Dependence Graph 91’

在gcc内部主要是在tree，gimple阶段，进行了dead code remove, use-def/def-use analysis, 以及许多在tree形态下需要多项式来完成的优化，在转化为ssa后，由于varriables都只有一个defination，所以只需要线性时间

gcc中ssa ir和tree是同时存在的，在转化为rtl后，就不存在ssa结构了，在regrename pass中，倒是对每一条insn，做了分析，由build_def_use function 生成def-use chain，

主要流程:

  • recog_insn_type
  • extract_insn
    • insn_extract
    • fill the recog_data, including operand nums, operand types and operand modes.
  • prepare_constraints
    • check each operands
  • scan_rtx
    • scan_rtx, if it’s insn type, call scan_rtx recursively, if it’s reg, call scan_rtx_reg, if it’s mem, call scan_rtx_address.
    • scan_rtx_reg, scan_rtx_address, check every rtx ( most are regs ), add new or close old chains created in previous stage.
